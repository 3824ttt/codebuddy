# 📚 读者荐读系统

读者通过点击投票推荐图书，达到阈值后自动入选采购书单，管理员据此向书商下单。

## 项目结构

```
├── backend/
│   └── app.py          # Flask 后端 + SQLite 数据库
├── frontend/
│   ├── index.html       # 读者端（浏览/搜索/荐读投票）
│   └── admin.html       # 管理后台（导入/书单/订单）
├── requirements.txt
├── start.bat            # Windows 一键启动
└── README.md
```

## 快速开始

### Windows
双击 `start.bat` 即可。

### 手动启动
```bash
pip install -r requirements.txt
python backend/app.py
```

## 访问地址

| 端 | 地址 |
|---|---|
| 读者端 | http://localhost:5000 |
| 管理后台 | http://localhost:5000/admin |

**管理员默认密码**: `admin123`（可在 `backend/app.py` 中修改 `ADMIN_PASSWORD`）

## 功能说明

### 读者端
- 浏览待选图书，按分类/搜索筛选
- 点击"推荐这本书"投票，支持热度排序
- 同一读者24小时内对同一本书只能推荐一次
- 实时查看每本书的荐读进度条

### 管理后台
- **仪表盘** — 系统概览统计
- **图书管理** — 查看/搜索/删除图书，批量 JSON 导入
- **荐读书单** — 达阈值的书单，可勾选生成采购订单，导出 CSV
- **采购订单** — 管理订单，确认到货
- **系统配置** — 调整推荐阈值、冷却时间

### 工作流
1. 管理员在后台上传候选图书
2. 读者浏览并点击推荐心仪图书
3. 点击量达到阈值 → 自动入选"荐读书单"
4. 管理员查看书单 → 勾选图书 → 生成向书商的采购订单
5. 到货后确认"已到货"
